# BLUEPRINT

How this site is styled and laid out — so you (future me, or a sibling) can rebuild the
same look on any other dataset without reverse-engineering it from scratch.

It's a single-page Next.js dashboard with a calm, editorial, "Linear/Vercel-ish" feel:
soft off-white (or near-black) background, one teal accent, thin borders, rounded cards,
monospace numbers, lowercase labels, and a confessional headline. Nothing flashy — the
restraint *is* the style.

---

## 1. The stack

| Piece | Choice | Why |
|---|---|---|
| Framework | **Next.js 15** (App Router, `app/`) | static-friendly, zero-config deploy on Vercel |
| Styling | **Tailwind CSS 3** | every style lives in `className`, no separate CSS files to hunt through |
| Charts | **Recharts** (bars/donut) + hand-rolled divs (funnel) | recharts for anything axis-based; plain divs when a bar chart is overkill |
| Icons | **lucide-react** | thin, consistent stroke icons that match the light UI |
| Theming | **next-themes** | light / dark / system toggle with no flash |
| Data | **static JSON** in `data/`, generated by `scripts/parse.py` from the Excel | UI never talks to a database; it imports JSON at build time |

Key idea: **the page is one big server component (`app/page.tsx`) that imports JSON and
hands it to one client component (`components/dashboard.tsx`).** Everything interactive
lives below `dashboard.tsx`. That split is what keeps it fast and simple.

## 2. The design system (copy these and you've got 80% of the look)

### Colors — semantic CSS variables, not raw Tailwind colors

The whole palette is six variables defined in `app/globals.css`, redefined under `.dark`,
and exposed to Tailwind in `tailwind.config.ts`. **Never write `bg-white` or `text-gray-500`
in a component** — always use these tokens so light/dark "just works":

```
bg        page background      light 252 252 250   dark 12 13 14
surface   cards / inputs       light 255 255 255   dark 22 23 25
border    every hairline       light 226 226 224   dark 38 40 43
fg        primary text         light 23 23 22      dark 240 240 238
muted     secondary text       light 115 115 110   dark 140 142 145
accent    the ONE red          light 220 38 38     dark 248 113 113
```

They're stored as raw RGB triples (`220 38 38`) so Tailwind's `<alpha-value>` works —
that's what lets you write `bg-accent/10`, `border-accent/40`, `text-accent`. One accent
color, used sparingly, carries the entire brand. Resist adding a second.

**Correct vs wrong stays single-accent on purpose.** A right answer lights up in the red
accent (`border-accent bg-accent/10 text-accent`); a wrong pick goes *neutral* — dimmed,
muted, `line-through` — never a second hue. Red is the brand, not an error color, so we
never pair it with green/rose; the contrast is accent-vs-absence, not red-vs-green.

### Typography

- **Sans** for everything, **mono** only via `tabular-nums` on numbers so columns align.
- Headline: `text-4xl md:text-5xl font-semibold tracking-tight leading-tight`.
- Section titles: `text-sm font-medium`. Section subtitles: `text-xs text-muted`.
- Micro-labels (filter row labels, "notes ·"): `text-[10px] uppercase tracking-wider text-muted`.
- **Labels are lowercase on purpose** ("share view", "year", "company", "clear"). It reads
  softer and more personal than Title Case. Keep it.

### Shape & spacing

- Cards: `rounded-xl border border-border bg-surface p-5` — memorize this, it's used ~10×.
- Pills / buttons / inputs: `rounded-full border border-border bg-surface`.
- Page width: `mx-auto max-w-6xl px-6`. Every section repeats this exact wrapper.
- Vertical rhythm between sections: `mt-10` (major) or `mt-4` (tight group).
- Hairlines everywhere; `*` gets `border-color: var(--border)` globally so a bare `border`
  class is always the right color.

### Motion

Almost none — just `transition-colors` on anything hoverable. Hover states nudge a border
to `accent/40` or text from `muted` to `fg`. No transforms, no big animations. The calm is
deliberate.

### Three signature touches

1. **`.grain` header background** (in `globals.css`): two faint red radial gradients. Gives
   the top of the page a subtle glow without an image.
2. **KPIs live in the header**: in the war-quiz build the live stats (score / streak / best /
   accuracy) sit inline in the top bar next to the brand — no separate stat strip — so the
   landing page is almost pure controls + question. Keep chrome minimal; the game is the page.
3. **Custom scrollbars + `::selection`** tinted with the accent. Small, but it's what makes
   it feel finished.

## 3. Layout, top to bottom

`components/dashboard.tsx` is the whole page. Sections in order, each in the `max-w-6xl` wrapper:

1. **Hero** (`.grain`, bottom border) — tiny breadcrumb row (icon · label · year range) on the
   left, `share view` button + theme toggle on the right; then the big confessional headline
   where the `<span className="text-muted">` / `text-accent` spans color-code the story
   (totals in muted, the win in teal), a muted sub-paragraph, and a heart-tagged dedication.
2. **Stat strip** — `grid grid-cols-2 md:grid-cols-4 gap-3` of `StatCard`s, pulled up with `-mt-6`.
   One card gets `highlight` (teal border + teal number).
3. **Positions + education** — `lg:grid-cols-5`, split 3/2.
4. **Funnel + by-year** — `lg:grid-cols-5`, split 2/3.
5. **Three category charts** — `md:grid-cols-2 lg:grid-cols-3` (outcome donut, by type, by source).
6. **Country grid** — clickable flags that set the country filter.
7. **The explorer** — heading + live result count, search input, filter chip rows, then the table.
8. **Footer** — one muted, slightly self-deprecating line.

The repeating recipe for a content block is always the same:

```tsx
<div className="rounded-xl border border-border bg-surface p-5">
  <h3 className="text-sm font-medium mb-1">Title</h3>
  <p className="text-xs text-muted mb-4">One-line explanation.</p>
  {/* chart or content */}
</div>
```

Internalize that card and the grid wrappers and you can lay out any new section in seconds.

## 4. How interactivity works (the filter pattern)

One `useState` object holds the entire UI state:

```ts
{ q, year, country, outcome, type, source, stage }
```

- **`useMemo` derives `filtered` rows** from `rows` + `filters` — no effects, no extra state.
- **Every filter control is a toggle**: clicking the active value sets it back to `null`
  (`prev[k] === v ? null : v`). Chips, the country grid, all of it. One mental model.
- **`share view`** serializes the filter object into URL query params and copies the link —
  that's the entire "sharing" feature, no backend.
- The table sorts client-side with a `sortKey` + `sortDir` and expands a notes row on click.

Want a new filter? Add a key to the state object, an `if` in the `useMemo`, and a `<FilterRow>`.
That's the whole contract.

## 5. Reusable building blocks (`components/`)

- **`stat-card.tsx`** — icon + label + big `tabular-nums` value + sub-line; optional `highlight`.
- **`flag.tsx`** — country name → flagcdn `<img>` (snaps to flagcdn's discrete 4:3 sizes;
  serves 2×/3× via `srcSet`); falls back to an emoji for Global/Remote/etc. Add countries to
  the `ISO` map.
- **`internship-table.tsx`** — sortable headers, expandable notes rows, and two tone maps
  (`STAGE_TONE`, `OUTCOME_TONE`) that map a status string → pill classes. This is where the
  secondary colors live; everything is an opacity tint of a named hue.
- **`filters.tsx`** — generic `FilterRow` (label + wrapping chips with counts).
- **`charts/`** — `funnel` (plain divs, accent bars fading by depth), `year-stack`,
  `category-bar`, `outcome-donut` (recharts; themed globally via the `.recharts-*` rules in
  `globals.css`, so chart text/grid automatically match the palette).
- **`theme-toggle.tsx`** — three-segment pill (sun / monitor / moon); guards against hydration
  flash by rendering a placeholder until `mounted`.

## 6. Data flow (so the look survives new data)

```
excel-data/*.xlsx   ← source of truth (NEVER committed/pushed)
      │  python scripts/parse.py   (npm run data)
      ▼
data/*.json         ← internships, stats, positions, education
      │  imported in app/page.tsx
      ▼
components/dashboard.tsx  ← renders everything
```

`scripts/parse.py` does the normalizing: it produces the `*Norm` fields
(`outcomeNorm`, `stageNorm`, …) that the filters and tone maps key off of, and pre-aggregates
all the `stats` (`funnel`, `byYear`, `byOutcome`, …) so the UI never computes anything heavy.
**If you add a chart, aggregate it in `parse.py` and read it from `stats`** — keep the
client dumb.

## 7. Rebuilding it for a different dataset — checklist

1. `npx create-next-app`, add Tailwind, recharts, lucide-react, next-themes.
2. Copy `app/globals.css` (tokens, grain, scrollbar, recharts theming) and `tailwind.config.ts`
   (the six color tokens + font families) verbatim. Change the accent triple if you want a
   different brand color.
3. Copy the `ThemeProvider` wiring in `app/layout.tsx`.
4. Recreate the section recipe: `max-w-6xl px-6` wrappers + the `rounded-xl border bg-surface p-5`
   card. Reuse `StatCard`, the `FilterRow`, and the filter-state pattern.
5. Shape your data into `data/*.json` with pre-aggregated `stats` and normalized `*Norm` fields.
6. Keep the rules: one accent, lowercase labels, `tabular-nums` on every number, thin borders,
   `transition-colors` only, and a headline that tells a human story instead of just stating
   metrics. That last one is what makes people actually read it.
